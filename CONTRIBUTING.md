# 反馈与贡献说明

感谢你帮助小隐寺提高内容质量。

## 可以提交什么

- 错别字、病句或失效链接；
- 可验证的事实性错误；
- 交易规则、监管政策或市场制度的变更；
- 更权威的一手资料；
- 可能让读者误解风险的表述。

请优先使用仓库的“内容纠错”Issue 模板，并提供：

1. 对应页面或文件；
2. 当前表述；
3. 建议修改；
4. 支持修改的原始来源；
5. 来源的发布日期或访问日期。

## 不接受什么

- 个股、期权或加密资产的买卖建议；
- 收益承诺和推广信息；
- 券商、交易所或代币的无关营销；
- 未经授权复制的文章、图片或付费资料；
- 用户账户、交易记录、API Key、助记词等敏感信息。

## Pull Request 与知识产权

小范围纠错可以提交 Pull Request。提交内容即表示：

- 你有权提交该内容；
- 内容不侵犯第三方权利；
- 你授权小隐寺永久、全球、免版税地使用、修改、发布并整合该贡献；
- 仓库其余内容仍适用 [专有内容许可](LICENSE.md)。

较大篇幅的原创内容或数据贡献，请先创建 Issue 与维护者确认授权方式。

## 行为准则

讨论应聚焦事实、来源与内容质量。禁止骚扰、人身攻击、欺诈、垃圾推广及泄露他人隐私。
